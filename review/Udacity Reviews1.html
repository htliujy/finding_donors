  • 随机森林

学生能够用清晰简洁的话来向一个没有机器学习或任何其他技术背景的人来解释最优模型的工作原理。

对随机森林的训练细节需要注意，在周志华对随机森林的讲解中(P178)：


每颗树对数据有放回的抽样，所抽出来的量(m)都必须与训练集的量相等，为了避免因抽样的量与训练的量不同所造成结果的偏差。而且这样的抽样又能够保证每次取出的数据大约是整个训练集的70%(在训练集出现过的数据)，这样使得每一棵树抽出来的数据都出现一定程度的差异性，使得训练出来的结果更加鲁棒。

随机森林对数据的选取是有放回的抽取，而特征抽取只取特征的子集而不是特征全集，这样做可以保证每棵树都不是有偏的，更多相关的知识可以参考这个blog

最终模型利用了网格搜索进行参数调优，至少挑战了一个参数，并且至少有三个可选值。如果模型参数不需要任何调整，学生需要给出明确的理由。

经过参数的调整模型得到很不错的提高，你可以优化make_score的函数编写，做到一步到位:

scorer = make_scorer(fbeta_score, beta=0.5)

为了使得到的结果更加准确可靠，你可以直接设置gridsearchvc里面的cv = 10，而不需要传kfold作为参数，理由如下：

  • cv在帮助文档的解释里面的说明:
    Determines the cross-validation splitting strategy. Possible inputs for cv are:
    • None, to use the default 3-fold cross validation,
    • integer, to specify the number of folds in a (Stratified)KFold,
    • An object to be used as a cross-validation generator.
    • An iterable yielding train, test splits

这里gridsearchcv默认使用一个叫分层抽样的kfold，与普通版的kfold相比，保证了训练数据分布的一致性，使实验结果更加稳定。而且设置更加方便，只要设置cv = 整数
